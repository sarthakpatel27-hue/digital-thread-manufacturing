# SOP — ECN Process (Portfolio Version)

## Purpose
Ensure product definition and manufacturing execution stay aligned through controlled change.

## Scope
EBOM, MBOM, routing, work instructions, inspection plans, tooling resources, controlled documents.

## Definitions
- **EBOM:** Engineering view (as-designed)
- **MBOM:** Manufacturing view (as-planned/as-built)
- **Baseline:** frozen snapshot of released definition
- **Effectivity:** date or serial range where change becomes active

## Procedure (high level)
1) Draft ECN with clear From→To and impacted items.
2) Cross-functional reviews (Eng, Mfg, Quality, Supply).
3) Approval & Release (new revisions + baseline).
4) Implementation (update MBOM/routing/WIs/resources; train if required).
5) Verification (trial/FAI/evidence attached).
6) Close ECN (only when execution matches released definition).

## Done rule
ECN is done when the shop can execute the new definition **without ambiguity**.
